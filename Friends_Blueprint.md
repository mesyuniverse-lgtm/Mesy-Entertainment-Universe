# Blueprint: Friends System

เอกสารนี้คือพิมพ์เขียว (Blueprint) เฉพาะสำหรับ **ระบบเพื่อน (Friends System)** ซึ่งเป็นส่วนหนึ่งของ Socialive ใน MESY Universe.

## 1. แนวคิดหลัก (Core Concept)

- **Private Connections:** สร้างพื้นที่สำหรับการเชื่อมต่อที่เป็นส่วนตัวและมีความหมายมากขึ้นระหว่างผู้ใช้.
- **Mutual Agreement:** เน้นย้ำว่าความเป็นเพื่อนต้องเกิดจากความยินยอมของทั้งสองฝ่าย ซึ่งแตกต่างจากการติดตาม (Following) ที่เป็นแบบทางเดียว.

## 2. ฟีเจอร์หลัก (Key Features)

- **Friend Request System:**
  - **การส่งคำขอ:** ผู้ใช้สามารถส่งคำขอเป็นเพื่อนไปยังผู้ใช้อื่นได้.
  - **การตอบรับ/ปฏิเสธ:** ผู้ใช้ที่ได้รับคำขอสามารถเลือกที่จะยอมรับหรือปฏิเสธคำขอนั้นๆ.
  - **การยกเลิกคำขอ:** ผู้ที่ส่งสามารถยกเลิกคำขอที่ยังไม่ได้รับการตอบรับได้.

- **Friend List:**
  - **แสดงรายชื่อเพื่อน:** หน้าสำหรับดูรายชื่อเพื่อนทั้งหมดที่เชื่อมต่อกันแล้ว.
  - **สถานะออนไลน์:** สามารถเห็นได้ว่าเพื่อนคนไหนกำลังออนไลน์อยู่.
  - **การค้นหาเพื่อน:** สามารถค้นหาเพื่อนจากในรายชื่อของตนเองได้.

- **Unfriending:**
  - ผู้ใช้สามารถยกเลิกการเป็นเพื่อนกับผู้ใช้อื่นได้ทุกเมื่อ ซึ่งการกระทำนี้จะมีผลกับทั้งสองฝ่าย.

## 3. ความแตกต่างระหว่าง Friends และ Followers

- **Friends (เพื่อน):** เป็นความสัมพันธ์แบบสองทาง (Mutual) ทั้งสองฝ่ายต้องยอมรับซึ่งกันและกัน เหมาะสำหรับการเชื่อมต่อส่วนตัว.
- **Followers (ผู้ติดตาม):** เป็นความสัมพันธ์แบบทางเดียว (One-way) ผู้ใช้สามารถติดตามใครก็ได้โดยไม่ต้องรอการอนุญาต เหมาะสำหรับบุคคลสาธารณะ, ศิลปิน, และ Creator.

## 4. หน้าและส่วนประกอบที่เกี่ยวข้อง (Key Pages & Components)

- **`/friends`:** หน้าหลักสำหรับจัดการทุกอย่างที่เกี่ยวกับเพื่อน ประกอบด้วย:
  - **Friend List:** แสดงรายชื่อเพื่อนทั้งหมด.
  - **Friend Requests:** แสดงคำขอเป็นเพื่อนที่เข้ามาใหม่.
  - **Suggested Friends:** ระบบแนะนำเพื่อนที่อาจจะรู้จัก.
  - **Search Friends:** เครื่องมือสำหรับค้นหาผู้ใช้เพื่อเพิ่มเป็นเพื่อน.
