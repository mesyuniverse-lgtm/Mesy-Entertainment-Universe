# Blueprint: Member Systems

เอกสารนี้คือพิมพ์เขียว (Blueprint) สำหรับหน้า **ระบบจัดการสมาชิก (Member Systems)** ซึ่งเป็นศูนย์กลางสำหรับผู้ดูแลระบบหรือสมาชิกที่มีสิทธิ์ในการตรวจสอบและจัดการฐานข้อมูลสมาชิกทั้งหมดในแต่ละระดับ ณ `/members/systems`

## 1. แนวคิดหลัก (Core Concept)

- **Central Database Hub:** เป็นศูนย์บัญชาการในการเข้าถึงฐานข้อมูลของสมาชิกในแต่ละระดับ (Level) ทำให้สามารถเห็นภาพรวมของโครงสร้างทั้งหมดและสถานะของแต่ละ Member ID ได้
- **Transparency & Activation:** สร้างความโปร่งใสโดยการแสดงให้เห็นถึงโครงสร้างรายได้ของทุกตำแหน่ง (Slot) ในระบบ และเป็นช่องทางให้สมาชิกใหม่สามารถเปิดใช้งาน (Activate) Member ID ที่ซื้อมาได้
- **Scalable Management:** ออกแบบให้สามารถรองรับการจัดการ Member ID จำนวนมาก (50,000+ IDs) โดยแบ่งตามระดับอย่างชัดเจน

## 2. โครงสร้างและหน้าต่างๆ (Structure & Pages)

### 2.1 หน้าหลัก (`/members/systems`)

- **เป้าหมาย:** เป็นหน้าสารบัญหลักที่นำทางไปยังฐานข้อมูลของแต่ละระดับ
- **องค์ประกอบหลัก:**
  - **Title:** "Members Database" หรือ "Member Systems"
  - **Status Checker:** เครื่องมือสำหรับค้นหา Member ID เพื่อตรวจสอบสถานะ, ระดับ, และรายได้โดยประมาณ
  - **Level Directory:** แสดงรายการของแต่ละระดับ (Level 0 - 50) ในรูปแบบของการ์ด
    - แต่ละการ์ดจะระบุ:
      - หมายเลขระดับ (เช่น "Level 0 Database")
      - ช่วงของ Member ID ที่อยู่ในระดับนั้น (เช่น "Member Slots: 1 - 999")
      - ปุ่ม "View Database" เพื่อนำทางไปยังหน้ารายละเอียดของระดับนั้นๆ
  - **Call-to-Action:** ปุ่ม "Purchase Member ID" สำหรับผู้ที่สนใจซื้อ Member ID เพิ่ม

### 2.2 หน้ารายละเอียดแต่ละระดับ (`/members/systems/level[X]`)

- **ตัวอย่าง:** `/members/systems/level0`, `/members/systems/level49`
- **เป้าหมาย:** แสดงฐานข้อมูลทั้งหมดของ Member ID ที่อยู่ในระดับนั้นๆ พร้อมสถานะและรายได้
- **องค์ประกอบหลัก:**
  - **Title:** ระบุระดับและช่วงของ ID อย่างชัดเจน (เช่น "Level 0 Members Database (1-999)")
  - **Activation Section:**
    - ส่วนสำหรับให้สมาชิกกรอก "Activation Code" ที่ได้รับหลังจากการซื้อ Member ID เพื่อเปิดใช้งานและผูก ID เข้ากับบัญชีของตนเอง
  - **Member Database Table:**
    - ตารางที่แสดงข้อมูลของทุก Member ID ในระดับนั้น
    - **คอลัมน์:**
      - **Member ID:** หมายเลข ID
      - **Name:** ชื่อ Avatar (แสดงเป็น "Waiting for member..." หากยังไม่มีเจ้าของ)
      - **Email:** อีเมลของเจ้าของ ID
      - **Level:** ระดับของ ID นั้นๆ
      - **Downline:** จำนวนสมาชิกในสายงาน (คำนวณตามสูตร)
      - **Income / Fee / Net-Income:** รายได้รวม, ค่าบริการ, และรายได้สุทธิ (คำนวณตามสูตร)
      - **Status:** สถานะ (เช่น Active/Claimed หรือ Inactive/Unclaimed) แสดงด้วยสีหรือไอคอนเพื่อให้เห็นภาพชัดเจน
  - **Pagination/Scrolling:** ระบบสำหรับการเลื่อนดูข้อมูลจำนวนมากในตาราง
